## Codebase Patterns
- Database schema changes require migrations in `MIGRATIONS` array in `src/storage/database.ts`
- When adding new database columns, update both `DatabaseSubject` and `SubjectInput` interfaces
- For API data extraction, update `convertSubjectToInput()` in `src/sync/syncService.ts`
- Component exports go in `src/components/index.ts`, utility exports in `src/utils/index.ts`
- React Native Image.prefetch() provides caching - no need for separate file system caching
- Tests with `upsertSubject()` need all fields from `SubjectInput` interface
- For radicals without Unicode characters, WaniKani provides images via `character_images` array

- **Files changed**:
  - `src/utils/answerValidation.ts` - Added getAllowedEditDistance() helper, integrated fuzzy matching into validateMeaningAnswer(), added isFuzzyMatch to MeaningValidationResult interface
  - `__tests__/utils/answerValidation.test.ts` - Added 40+ new tests for typo forgiveness scenarios
- **Learnings for future iterations:**
  - Typo tolerance is based on answer length: 1-3 chars require exact match, 4-6 chars allow 1 edit, 7+ chars allow 2 edits
  - Exact matches are checked first, then fuzzy matches - this ensures exact matches don't get isFuzzyMatch flag
  - Blacklist checking uses exact match only (not fuzzy) to avoid false rejections
  - The isFuzzyMatch flag allows UI to show different feedback (yellow vs green) in future stories
