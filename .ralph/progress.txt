## Codebase Patterns
- SRS levels map: Apprentice (stages 1-4), Guru (5-6), Master (7), Enlightened (8), Burned (9)
- SRS level colors and utilities are in `src/theme/colors.ts` and exported via `src/theme/index.ts`
- ReviewItem interface in ReviewSession.tsx includes `srsStage: number`
- ReviewsScreen.tsx converts DatabaseAssignment.srs_stage to ReviewItem.srsStage in subjectToReviewItem()
- Badge-style components use position absolute with top/right for header placement
- react-native-reanimated requires: babel plugin, jest mocks (both reanimated and worklets), all hooks called before early returns
- Reduced motion: Use AccessibilityInfo.isReduceMotionEnabled() + addEventListener('reduceMotionChanged') to respect accessibility settings
- useEffect dependencies: Don't include reanimated shared values in deps - mocks create new objects causing effect to re-run (use eslint-disable comment)
- Database CRUD: Follow pattern of XxxInput interface for inserts, Database interface for queries. Use INSERT OR REPLACE for upserts, INSERT OR IGNORE for duplicate prevention
- Synonym storage: user_synonyms (synced to WaniKani) and pending_synonyms (awaiting sync) tables. Use getUserSynonymsBySubjectId() to get synonyms for validation
