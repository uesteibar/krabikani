## Codebase Patterns
- Quiz components (LessonQuiz, ReviewSession) have nearly identical UIs but different question ordering logic
- Question types are 'meaning' or 'reading', stored in QuestionType/ReviewQuestionType
- Subject colors use COLORS.neutral.black and COLORS.neutral.white from theme/colors.ts
- Styles use StyleSheet.create with COLORS, SPACING, BORDER_RADIUS from theme
- Test files use testID attributes for querying elements
- Pre-existing test failures exist in progress text format assertions and SettingsScreen tests (unrelated to new features)
- PanResponder gesture state cannot be unit tested by directly calling handlers; use structural tests to verify handlers exist, functional tests require E2E or manual verification

## 2026-01-25 - US-003
- **What was implemented**: Created MnemonicText component for parsing and rendering mnemonic tags
  - New MnemonicText component parses `<radical>`, `<kanji>`, and `<vocabulary>` tags
  - Radical text highlighted with blue color (#00AAFF / SUBJECT_COLORS.radical)
  - Kanji text highlighted with pink color (#FF00AA / SUBJECT_COLORS.kanji)
  - Vocabulary text highlighted with purple color (#AA00FF / SUBJECT_COLORS.vocabulary)
  - Highlighted text also rendered with bold font weight
  - Malformed, unclosed, mismatched, and unknown tags render as plain text (graceful handling)
  - Regular text between tags renders normally
- **Files changed**:
  - src/components/MnemonicText.tsx (new component with parseMnemonicText function)
  - __tests__/components/MnemonicText.test.tsx (25 comprehensive tests)
- **Learnings for future iterations**:
  - SUBJECT_COLORS from theme/colors.ts already defines exact WaniKani colors for radical/kanji/vocabulary
  - React Native style arrays (using `style={[a, b]}`) require test assertions with expect.arrayContaining([expect.objectContaining({...})])
  - Regex with backreference `\1` ensures matching opening/closing tags (prevents `<radical>...</kanji>` from matching)
  - Global regex in JS needs `lastIndex` reset when reusing the pattern

## 2026-01-25 - US-006
- **What was implemented**: Added swipe gestures to lesson cards for navigation
  - Swipe left advances to next item (same as tapping Next)
  - Swipe right goes back to previous item (same as tapping Back)
  - Swipe right on first item does nothing (isFirstItem check)
  - Swipe left on last item completes the batch (calls onBatchComplete)
  - SWIPE_THRESHOLD of 50 pixels to avoid accidental triggers
  - SWIPE_VERTICAL_THRESHOLD of 100 pixels to distinguish from scrolling
  - Button navigation continues to work alongside swipe gestures
- **Files changed**:

  - __tests__/components/LessonBatch.test.tsx (updated tests - structural tests verify swipe area exists and has handlers)
- **Learnings for future iterations**:

  - Structural tests verify swipe area exists and has handlers attached; functional swipe testing requires manual browser verification or E2E tests
