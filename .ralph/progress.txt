## Codebase Patterns
- React Native 0.83.1 with TypeScript (uses @react-native/typescript-config)
- Jest requires transformIgnorePatterns for @react-navigation and other ES module packages
- Folder structure: src/screens, src/components, src/api, src/storage, src/utils, src/navigation
- Navigation uses @react-navigation/native with native-stack navigator
- Quality checks: npm run typecheck, npm run lint, npm test
- Use react-native-keychain for secure storage (API keys, tokens)
- Use @testing-library/react-native for component tests
- Mock native modules in __mocks__/ directory (e.g., react-native-keychain.ts)
- Export modules via index.ts barrel files in each directory
- WaniKani API: use `Wanikani-Revision: 20170710` and `Authorization: Bearer <token>` headers
- Mock global fetch in tests: `(globalThis as { fetch: typeof fetch }).fetch = mockFetch`
- API types are in src/api/types.ts, API client logic in src/api/wanikaniApi.ts
- WaniKaniClient class handles authentication and typed requests
- WaniKaniApiError has code (ApiErrorCode), statusCode, message, and retryable properties
