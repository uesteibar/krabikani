# Ralph Progress Log

## Codebase Patterns

- Custom fonts go in `assets/fonts/` and are linked via `react-native.config.js`
- Theme exports are centralized in `src/theme/index.ts`
- Use Platform.select() for platform-specific font family names (iOS uses font family name, Android uses file name without extension)
- DATABASE_VERSION in database.ts must match the expected version in tests
- Tests with state transitions after fireEvent.press() may need `findByTestId` with async await
- searchSubjects(query, hiraganaQuery, limit) returns learned items - caller should pre-convert romaji to hiragana using romajiToHiragana() from src/utils
- `getPracticeItems(limit)` returns random learned items (srs_stage >= 5) excluding currently pending reviews
- `subjectToReviewItem()` is duplicated per-screen (ReviewsScreen, PracticeScreen) ŌĆö consider extracting if a third use appears
- Quiz questions are shuffled ŌĆö tests rendering LessonQuiz must handle either meaning or reading first; use `submitWrongAnswer` helper for question-type-agnostic wrong answer submission
- Use React `key` prop to force component remount when scroll position needs resetting (e.g., `key={item.id}` on cards with ScrollView content)
- Completion screens (LessonCompletion, ReviewCompletion) share the same animated pattern: animated checkmark, staggered fade-in (200ms delay, 500ms duration), confetti for perfect sessions, encouragement messages, and "Synced"/"Queued" badge
