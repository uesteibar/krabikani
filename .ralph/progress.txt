# Ralph Progress Log

## Codebase Patterns

- Import icons as `{ MaterialDesignIcons } from '@react-native-vector-icons/material-design-icons'` (named export, NOT default from @expo/vector-icons)
- Custom fonts go in `assets/fonts/` and are linked via `react-native.config.js`
- Theme exports are centralized in `src/theme/index.ts`
- Use Platform.select() for platform-specific font family names (iOS uses font family name, Android uses file name without extension)
- DATABASE_VERSION in database.ts must match the expected version in tests
- Tests with state transitions after fireEvent.press() may need `findByTestId` with async await
- searchSubjects(query, hiraganaQuery, limit) returns learned items - caller should pre-convert romaji to hiragana using romajiToHiragana() from src/utils
- `getPracticeItems(limit)` returns random learned items (srs_stage >= 5) excluding currently pending reviews
- `subjectToReviewItem()` is duplicated per-screen (ReviewsScreen, PracticeScreen) — consider extracting if a third use appears
- Quiz questions are shuffled — tests rendering LessonQuiz must handle either meaning or reading first; use `submitWrongAnswer` helper for question-type-agnostic wrong answer submission
- Use React `key` prop to force component remount when scroll position needs resetting (e.g., `key={item.id}` on cards with ScrollView content)
- Completion screens (LessonCompletion, ReviewCompletion) share the same animated pattern: animated checkmark, staggered fade-in (200ms delay, 500ms duration), confetti for perfect sessions, encouragement messages, and "Synced"/"Queued" badge
- Components using `useTheme()` require `<ThemeProvider forcedColorScheme="light">` wrapper in tests (see LevelIndicator, HomeScreen tests)
- Components using `useSafeAreaInsets()` require `<SafeAreaProvider initialMetrics={...}>` wrapper in tests (see InstructionsScreen tests)
- The react-native-reanimated mock in jest.setup.js must export any Animated.* components used (View, Text, Image) — add new ones as needed
- Prefer `getByText(/regex/)` over `.props.children` assertions for text content — resilient to nested `<Text>` element changes
- `jest.clearAllMocks()` resets mock return values — re-setup `mockResolvedValue` after calling it
- SettingsScreen save button is disabled when `apiKey.trim() === storedApiKey` — empty key error path is unreachable via UI
- jest.config.js has `testPathIgnorePatterns` for `.ralph/.worktrees/` to exclude old worktree copies from test discovery
- `addUserSynonym` uses `INSERT OR REPLACE` but SQLite UNIQUE is case-sensitive — for case-insensitive updates, delete existing row first then insert
- Background sync functions return result objects with `success`, `syncedCount`, etc. and are composed in `backgroundSync()` function
- Custom hooks live in src/hooks/, tests in __tests__/hooks/
- When refactoring to use shared components, testIDs change: ProgressHeader uses `progress-header-*`, SubjectDisplay uses `subject-display-*`, IncorrectFeedbackView prefixes all testIDs with the `testID` prop value
- ReviewSession tests now use ProgressHeader testIDs (progress-header-count, progress-header-remaining, progress-header-wrap-up, progress-header-progress) and SubjectDisplay testIDs (subject-display-text, subject-display-feedback-label, srs-badge)
- Quiz engine types live in src/components/quiz/types.ts; answer validation in src/components/quiz/answerValidation.ts — use `validateInput` for pre-submission checks and `validateAnswer` for correctness evaluation
- Unified Question type has questionType: 'meaning' | 'reading' | 'reverse' — reverse mode uses exact displayText match with no fuzzy matching
- QuizEngine component lives in src/components/quiz/QuizEngine.tsx — config-driven via QuizEngineConfig, wrapper screens handle domain logic (SRS, navigation, data loading) via callbacks
- QuizEngine tests: don't mock useQuestionInput with React.useState in jest.mock factory — use the real hook or require('react') inside the factory
- Reverse mode Question: `displayText` = English meaning (shown on screen), `correctAnswers` = [Japanese characters] — answerValidation uses `correctAnswers[0]` for comparison, NOT `displayText`
- App state sync module (appStateSync.ts) uses listener pattern with Set + add/remove functions for sync status, sync errors, and background sync completion
- Cold start sync bypasses throttle check and is triggered via `setImmediate()` in `initializeAppStateSync()` to allow initialization to complete first
- Throttle logic uses `getSyncStatus()` from database to get last sync timestamps and compares against SYNC_THROTTLE_MS (5 minutes)
- react-native-reanimated mock in jest.setup.js needs `cancelAnimation` and `withRepeat` for animated components that use indeterminate loops
